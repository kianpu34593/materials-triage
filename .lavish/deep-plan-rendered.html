Materials-Triage Agent

A public-data-only agent that turns a scientist's request into a ranked, fully-cited shortlist with caveats and clearly-marked missing data, in two views (PI summary / technical audit). No wet-lab actions, no private data, no paywalled scraping — and prompt-injection-resistant by construction. 

"Find promising oxide dielectric candidates for thin-film experiments.
 Prefer stable, wide band gap, non-toxic, simple compositions, public
 evidence. Return a ranked shortlist with caveats."

The load-bearing decision

The LLM never invents scientific facts. Public databases supply every number (tagged with provenance); deterministic code filters and ranks; the LLM only builds the spec, proposes hypotheses, and writes grounded, cited narrative.

Three properties fall out of this: traceable (every run is a recorded, replayable TriageRun), configurable (tweak a knob, resume from that step), and generalizable with zero setup (no mandatory profile — the spec is LLM-built and remembered).

Workflow walkthrough

1 · Input policy gate

Allowlist; refuse forbidden/out-of-scope → logged refusal. Not recorded into the run.

2 · Spec building (LLM + human)

Convert → Filling Spec → on missing fields, Recommend LLM (seeded by defaults + lab memory) → User input accepts or fills in (manual fills pass a user-input gate) → Final Spec.

3 · Hypothesis (LLM + RAG)

Propose candidates / property ranges from the literature. Hypotheses, not facts.

4 · Retrieve (code, not LLM)

Deterministic API calls — v1: Materials Project only. Returns candidates + PropertyValues, each with Provenance.

5 · Hard filters

Drop any candidate violating any hard constraint in the spec; each drop records a reason.

6 · Ranking

v1: weighted average of target properties; applies on_missing and flags missing data. (Pareto/down-screening later.)

7 · Synthesis (LLM + RAG)

Grounded narrative + mechanistic "why," each claim cited; explains the excluded set. No invented numbers.

8 · Output validator

Every referenced ID + citation must resolve to retrieved provenance; ungrounded → reject & retry.

9 · Render

view=pi concise summary · view=audit renders the full trace.

Cross-cutting

Literature RAG (untrusted) feeds steps 3 & 7. Two buckets: lab memory (specs) + TriageRun (trace); trace saves to memory.

Verification

  • Per increment: pytest tests/<file> green + ruff check clean.
  • Deterministic core: scoring/filter/rank/provenance/trace on fixtures, exact assertions, no network/LLM.
  • Behavioral (mock LLM, temp=0): forbidden ask → refusal; injected abstract changes nothing; fabricated ID rejected.
  • End-to-end: --view pi and --view audit both produce ranked, cited shortlists with missing flags; resume --set weights.band_gap=0.4 --from rank reuses cached retrieval.
  • Scorecard: python eval/run_eval.py.

Why it's safe & honest

  • Facts from tools, not the LLM — renderer fills numbers; hallucination is structurally impossible in the numeric layer.
  • Capability by construction — no wet-lab/private-DB/scraper tool exists.
  • Trust boundary — retrieved text is untrusted DATA, never instructions.
  • Output validator — IDs + citations must resolve; no ungrounded numbers.
  • Missing data first-class — ranked-but-flagged, never silently dropped or guessed.
  • No DB to host — HTTP client over public APIs; only local state is run traces + memory.
